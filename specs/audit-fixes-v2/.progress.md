## Learnings
- Command sanitizer is defense-in-depth: qualityCommands and verify commands are set by the coordinator, so they're "trusted at write time." Sanitizer catches file tampering.
- macOS doesn't ship `flock` -- use mkdir-based locking as fallback for cross-platform compatibility.
- `_build_groups_from_predefined` mutates task['dependencies'] in-place because it assigns to the list directly (line 730-731). Fix requires shallow copy of the task dict, not just the deps list.
- merge.md Step 5 (pre-merge conflict detection) currently runs AFTER Step 4 (worktree merge), which defeats its purpose. Simple renumber fixes this.
- hooks.json already registers merge-guard.sh and teammate-idle-gate.sh hooks, but SKILL.md's hooks table only documents 4 of the 6 hooks.
- Counter files in dispatch-coordinator.sh and teammate-idle-gate.sh use predictable names, but since SPEC_NAME and SESSION_ID are now sanitized via _sanitize_name(), the symlink attack vector is mitigated.
- validate-pre-merge.py quality loop checks build/test/lint but skips typecheck -- easy one-line fix to add it.
- All Python scripts follow same pattern: argparse + json.load + json.dump. None have try/except around main().
